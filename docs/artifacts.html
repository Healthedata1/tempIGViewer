This page provides a list of the FHIR artifacts defined as part of this implementation guide.

Behavior: Search Parameters

These define the properties by which a RESTful server can be searched. They can also be used for sorting and including related resources.

GroupValueReference

Reference value held by characteristic

Structures: Resource Profiles

These define constraints on FHIR resources that need to be complied with by conformant implementations

Argonaut Patient List (Group) Profile

Profile on the R4 Group resource to for Argonaut Patient List Project. Note that the Group resource is used for this patient list use case even though there is also a FHIR List resource
Argonaut Patient List (Group) Profile

Profile on the R4 Group resource to for Argonaut Patient List Project. Note that the Group resource is used for this patient list use case even though there is also a FHIR List resource

Structures: Extension Definitions

These define constraints on FHIR data types that need to be complied with by conformant implementations

DEQM Measure Scoring Extension

Indicates how the calculation is performed for the measure, including proportion, ratio, continuous-variable, and cohort. The value set is extensible, allowing additional measure scoring types to be represented. It is expected to be the same as the scoring element on the referenced Measure
DEQM Submit Data Update Type Extension

​This Extension supports the concepts ‘incremental’ and ‘snapshot’. The DEQM Consumer Server SHALL use this Extension to advertise whether it supports snapshot or incremental updates for the DEQM Submit Data transaction. The DEQM Producer SHALL use this Extension to indicate whether the payload is a snapshot or incremental update for Submit Data transactiosn.
Argonaut Patient List Questionnaire

A reference to a form definition that is bound to certain types of lists and defines each column for the purpose of providing additional data for each member
Argonaut Patient List QuestionnaireResponse

A reference to a QuestionnaireResponse of prepopulated data based on a Patient List Questionnaire that is bound to certain types of lists and provides additional data for a Group.member

Terminology: Value Sets

These define sets of codes used by systems conforming with this implementation guide

Argonaut Group Characteristic Value Set
Used to define group characteristic and limited to ‘location’ ….
DEQM Update Type Value Set

Concepts for how a DEQM Consumer supports data exchange updates. The choices are snapshot or incremental updates

Terminology: Code Systems

These define new code systems used by systems conforming with this implementation guide

Argonaut Group Characteristic Code System
Used to define group characteristic and limited to ‘location’ ‘team’ ‘organization’ ‘practitioner’
DEQM Update Type Code System

Concepts for how a DEQM Consumer supports data exchange updates. The choices are snapshot or incremental updates
